# JoJoMath 业务主链路

使用这份文档时，要把设计决策放回产品行为里判断。下面是第一版工作骨架，在把它当作最终规范前，需要用产品负责人意见和真实 Figma 流程确认。

## 用户角色

- **学生**：完成学习任务、练习数学、获得反馈、查看进步，并回到下一次学习。
- **家长**：理解孩子的学习状态、评估产品价值、接收建议，并可能做出续学或付费决策。
- **老师或运营人员**：待补充：确认这个角色是否在 JoJoMath 当前产品范围内。
- **营销访客**：看到一个学习承诺，理解产品价值，并完成转化动作。

## 学生学习链路

1. **进入任务**：学生打开被分配的任务、推荐练习、诊断测评或活动。
2. **理解目标**：学生看到本次目标、进度、难度和下一步动作。
3. **完成练习**：学生答题，页面需要给数学内容和输入区域足够空间。
4. **获得反馈**：学生看到对错、解析、提示或重试引导。
5. **看到进步**：学生看到完成度、连续学习、掌握度或奖励状态。
6. **进入下一步**：学生继续练习、复盘错题、开始新任务，或清楚地结束本次学习。

设计含义：练习循环应该稳定、清晰、有激励感。避免为了装饰而压缩题目、输入区或反馈区。

## 家长信任链路

1. **快速扫读状态**：家长能快速知道孩子做了什么、是否有进步。
2. **看到依据**：家长能理解正确率、用时、薄弱点、进度或作答示例。
3. **理解含义**：产品用普通语言解释这些数据意味着什么。
4. **获得建议**：家长知道下一步该继续练习、复习薄弱点、升级、预约或分享。
5. **做出决策**：家长有足够信心继续使用或付费。

设计含义：家长端 UI 要优先保证信息层级、可信度和解释清楚，不要过度追求可爱或热闹。

## 转化链路

1. **吸引注意**：用户看到具体的数学学习收益或及时优惠。
2. **建立相关性**：信息与年龄、年级、痛点、季节或学习目标相关。
3. **建立信任**：视觉或文案展示真实产品、进步依据、用户证明或可信机制。
4. **推动行动**：行动按钮或行动指令清楚，并且与当前活动目标一致。

设计含义：宣传视觉不能脱离真实产品。能用真实 UI 或产品证据时，优先使用真实证据。
