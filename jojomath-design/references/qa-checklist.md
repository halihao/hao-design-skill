# JoJoMath 设计 QA 检查清单

完成 UI、宣传视觉或设计审核前，必须运行对应检查清单。

## UI 检查清单

- 是否明确目标用户角色：学生、家长、老师/运营人员或营销访客。
- 是否服务 `business-flows.md` 中的某个明确业务链路。
- 主操作是否清楚、可见、无歧义。
- 数学内容是否仍然易读，并且处于视觉中心。
- 是否覆盖必要反馈状态：成功、错误、提示、加载、空状态和完成状态。
- 学生端语气是否鼓励、具体。
- 家长端信息是否可信、易扫读、有依据。
- 布局是否适配请求中的设备尺寸。
- 组件是否符合 `design-system.md`，或清楚说明了临时假设。
- 是否没有把占位规范当成最终品牌规则。

## 宣传视觉检查清单

- 是否清楚说明受众、渠道、格式和行动按钮。
- 主信息是否具体，并且在目标尺寸下可读。
- 视觉是否通过 UI、数学内容、学习证据或官方品牌资产体现产品相关性。
- 宣传主张是否具体，并且没有过度承诺。
- 层级是否只有一个主信息、一个视觉锚点和一个行动指令。
- 是否已经考虑 logo、二维码、产品截图、法律文字等必需元素。
- 是否避免了没有 JoJoMath 依据的泛教育视觉。
- 缺失信息是否被列为假设。

## 设计审核检查清单

- 每条问题是否对应到 JoJoMath 的原则、链路、页面模式或检查项。
- 是否先列高影响业务/可用性问题，再列视觉细节问题。
- 是否区分“已确认违规”和“需要更多品牌证据确认”的问题。
- 每条问题是否给出具体修改建议。
- 是否说明了哪些必要素材没有拿到。

## 最终回复检查清单

- 总结这次设计或审核完成了什么。
- 说明使用了哪些 reference。
- 说明仍然存在的假设或缺失素材。
- 如果涉及实现工作，说明修改了哪些文件以及如何验证。
