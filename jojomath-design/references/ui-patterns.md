# JoJoMath UI 页面模式

这些模式用于作为页面设计的起点。后续请用截图或 Figma 链接替换占位内容，形成正式标准。

## 学生做题页

- **目标**：让学生专注在当前数学题上，并清楚知道下一步动作。
- **必要层级**：进度/上下文、题目、答题输入、反馈/提示、主操作。
- **必须设计的状态**：初始、答题中、提交后答对、提交后答错、展开提示、展开解析、加载中、完成。
- **避免**：营销式大横幅、把题目挤到下方的装饰卡片、带惩罚感的错误反馈。
- **待补充证据**：补充官方认可的做题页截图或 Figma。

## 学习结果页

- **目标**：给学习闭环、强化进步感，并引导下一步有价值的动作。
- **必要层级**：结果总结、关键表现信号、具体提升建议、下一步操作。
- **学生版**：更有激励感、庆祝感和行动导向。
- **家长版**：更解释性、更重依据。
- **待补充证据**：补充结果页案例和奖励强度规则。

## 错题复盘或解析页

- **目标**：把错误转化成可理解的下一步。
- **必要层级**：原题、学生答案、正确思路、误区或提示、重试/复习动作。
- **文案规则**：具体说明该改进什么，不要羞辱或打击学生。
- **待补充证据**：补充解析、提示和重试模式案例。

## 家长报告页

- **目标**：帮助家长快速理解状态、进步、薄弱点和建议动作。
- **必要层级**：总结、关键指标、能力点拆解、解释、建议、行动按钮。
- **视觉规则**：使用冷静的信息设计；只有真正有意义的进步时才使用强庆祝视觉。
- **待补充证据**：补充报告/看板案例和官方指标定义。

## 新用户引导页

- **目标**：只收集个性化学习或快速开始所必需的信息。
- **必要层级**：每一步一个明确任务、可见进度、低摩擦选择、必要的安心说明。
- **待补充证据**：补充引导步骤、用户字段和文案限制。

## 活动页或专题页

- **目标**：把活动承诺连接到一个立即可执行的产品动作。
- **必要层级**：活动收益、产品证据或预览、参与机制、行动按钮。
- **视觉规则**：活动页可以比核心工具更有表现力，但仍然必须像 JoJoMath。
- **待补充证据**：补充活动落地页或季节性活动参考。

## 响应式规则

- **移动端**：每屏优先承载一个主要任务；不要把密集报告模块硬塞进小卡片。
- **桌面端**：额外宽度应用于比较、证据或辅助上下文，不要只制造空白装饰。
- **平板端**：保留舒适触控和可读数学内容。
- 待补充：补充优先支持设备和最小视口假设。
