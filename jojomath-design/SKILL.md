# JoJoMath 设计规范

## 概览

使用这个 skill 时，目标是产出符合 JoJoMath 产品气质的 UI 和宣传视觉。`references/` 里的文件是主要依据；如果某些内容仍然是占位或待补充，不要把它当作已确认的品牌规则，要在输出中明确说明这是待确认假设。

## 根据任务选择阅读资料

- **UI 设计或改版**：阅读 `references/product-principles.md`、`references/business-flows.md`、`references/design-system.md`、`references/ui-patterns.md` 和 `references/qa-checklist.md`。
- **宣传图、海报、社媒图或活动图**：阅读 `references/product-principles.md`、`references/marketing-visuals.md`、`references/examples.md` 和 `references/qa-checklist.md`。
- **设计审核或走查**：阅读对应的 UI 或宣传图资料，再阅读 `references/qa-checklist.md`；优先指出具体问题和修改建议。
- **Figma、截图或已有素材对齐**：先检查用户提供的设计稿或图片，再对照 `references/examples.md` 和相关规范文件判断是否符合 JoJoMath。

## 执行规则

1. 先确认这次设计服务的是哪个用户角色、哪个业务目标、哪个主链路步骤，再做视觉决策。
2. 按任务选择需要的 reference 文件，不要为了小任务加载无关资料。
3. 明确区分学生端、家长端、转化/营销场景，不要混用同一套表达方式。
4. 如果 reference 里仍写着 `待补充` 或 `待确认`，不要把它包装成最终规范；要标注为假设。
5. 输出尽量具体：页面结构、组件选择、层级关系、视觉方向、文案块、交互状态和 QA 结论。
6. 做宣传图时，优先使用 `assets/brand/` 的品牌资产、`assets/screenshots/` 的产品截图、`assets/figma-exports/` 的 Figma 导出图，以及 `assets/marketing-examples/` 的过往宣传案例。

## 输出要求

- **UI 设计**：说明目标用户、业务目标、页面层级、组件/状态决策、文案语气，以及必要的响应式或交互说明。
- **宣传视觉**：说明主推信息、目标受众、版式概念、视觉层级、所需素材、文案方向和使用/导出场景。
- **设计审核**：区分高影响问题和视觉细节问题，指出违反了哪条 JoJoMath 原则或模式，并给出可执行修改建议。
- **完成前**：用 `references/qa-checklist.md` 做自检，并列出仍未确认的假设。

## 资料索引

- `references/product-principles.md`：产品定位、目标用户、品牌气质和设计原则。
- `references/business-flows.md`：学生、家长、学习、报告和转化相关的核心链路。
- `references/design-system.md`：颜色、字体、间距、组件、状态、图标和插画规则。
- `references/ui-patterns.md`：学习页、做题页、结果页、报告页、引导页、活动页等页面模式。
- `references/marketing-visuals.md`：宣传图策略、构图、文案语气、视觉层级和禁忌。
- `references/examples.md`：优秀案例、反例，以及用于持续完善规范的证据库。
- `references/qa-checklist.md`：UI、宣传视觉和设计审核的最终检查清单。
